## Context

当前管理员审核页面 (`/admin/products/[id]/review`) 使用 `review-client.tsx` 在网格视图中展示缩略图，通过 hover 显示操作按钮（通过/驳回/重新生成）。图片尺寸小，无法检查纹路、纽扣、领口等细节，也无法与原图对比。审核效率低，且驳回时没有理由记录。

现有技术栈：Next.js App Router + React + Tailwind CSS + shadcn/ui + Drizzle ORM。

## Goals / Non-Goals

**Goals:**
- 提供全屏审核弹窗，支持大图查看和缩放
- 支持 AI 生成图与原图的并排对比
- 提供预设驳回理由 + 自定义输入
- 审核完成后自动切换到下一张待审核图
- 支持键盘快捷键提升操作效率
- 在数据库中持久化驳回理由

**Non-Goals:**
- 不重构现有的网格视图和批量操作逻辑
- 不修改审核状态的业务流程（pending/approved/rejected/regenerate 保持不变）
- 不添加图片编辑/裁切功能
- 不改变交付流程

## Decisions

### 1. 弹窗实现方案：shadcn/ui Dialog（全屏模式）

使用 shadcn/ui 的 `Dialog` 组件，设置为近全屏尺寸（`max-w-[95vw] max-h-[95vh]`）。

**替代方案**: 使用独立的全屏页面路由。但弹窗保持在审核列表页面的上下文中，关闭后可立即回到列表，体验更连贯。

### 2. 图片缩放方案：CSS transform + 鼠标/手势控制

使用 CSS `transform: scale()` + `translate()` 实现缩放和拖拽平移，不引入额外依赖。

- 鼠标滚轮缩放
- 拖拽平移
- 双击重置/放大
- 缩放级别按钮（适应窗口 / 100% / 200%）

**替代方案**: 使用 `react-zoom-pan-pinch` 库。但该功能不复杂，自行实现可避免额外依赖，且更灵活。

### 3. 布局设计：左右分栏

弹窗内部布局：
```
┌──────────────────────────────────────────────────────────┐
│  [← 上一张]   3/20 待审核   [下一张 →]  缩放控制   [×]   │
├──────────────────────────────────┬───────────────────────┤
│                                  │ 原图参考 (3)          │
│                                  │                       │
│     AI 生成图（大图区域）          │  ┌─────────────────┐ │
│     支持缩放、拖拽、双击放大       │  │  原图1 (大图)    │ │
│                                  │  │  点击放大查看     │ │
│                                  │  └─────────────────┘ │
│                                  │  ┌─────────────────┐ │
│                                  │  │  原图2 (大图)    │ │
│                                  │  │  点击放大查看     │ │
│                                  │  └─────────────────┘ │
│                                  │  (可垂直滚动)         │
├──────────────────────────────────┴───────────────────────┤
│  [✓ 通过]  [✗ 驳回 ▼]  [↻ 重新生成]                      │
│                                                          │
│  驳回理由（展开时显示）：                                   │
│  ☐ 裁切问题  ☐ 模特不自然  ☐ 纹路偏差                     │
│  ☐ 颜色偏差  ☐ 背景不合适  ☐ 变形/畸变                    │
│  ☐ 分辨率低  ☐ 光影不自然  [自定义输入...]                  │
└──────────────────────────────────────────────────────────┘
```

**替代方案（已弃用）**: 上下分区布局（上方生成图，下方原图水平缩略图条）。实际使用中发现底部缩略图太小，不便于对比纹路和细节，因此改为左右分栏，右侧原图以较大尺寸纵向排列。

### 4. 预设驳回理由：前端枚举 + 数据库存储

预设理由在前端定义为常量数组，驳回时可多选 + 自定义输入。最终以 JSON 字符串存储到 `rejectionReason` 字段。

```typescript
const REJECTION_REASONS = [
  { id: 'crop_issue', label: '裁切问题' },
  { id: 'unnatural_model', label: '模特不自然' },
  { id: 'texture_mismatch', label: '纹路偏差' },
  { id: 'color_deviation', label: '颜色偏差' },
  { id: 'bad_background', label: '背景不合适' },
  { id: 'distortion', label: '变形/畸变' },
  { id: 'low_resolution', label: '分辨率低' },
  { id: 'unnatural_lighting', label: '光影不自然' },
  { id: 'wrong_pose', label: '姿势不当' },
  { id: 'accessory_error', label: '配饰错误' },
  { id: 'fabric_error', label: '面料质感错误' },
  { id: 'detail_missing', label: '细节丢失（纽扣/拉链/口袋等）' },
  { id: 'proportion_issue', label: '比例失调' },
  { id: 'watermark_artifact', label: '水印/伪影' },
] as const;
```

### 5. 自动翻页策略

审核操作（通过/驳回/重新生成）后：
- 自动切换到下一张 **待审核** 图片
- 如果当前是最后一张待审核图，显示 "审核完成" 提示并关闭弹窗
- 保持平滑过渡动画（fade/slide）

### 6. 键盘快捷键

| 快捷键 | 操作 |
|--------|------|
| `A` / `Enter` | 通过 |
| `R` | 驳回（展开理由面板） |
| `←` / `→` | 上一张 / 下一张 |
| `+` / `-` | 放大 / 缩小 |
| `0` | 重置缩放 |
| `Esc` | 关闭弹窗 |

### 7. 数据库迁移

在 `product_generated_images` 表新增：
```sql
ALTER TABLE product_generated_images ADD COLUMN rejection_reason text;
```

使用 Drizzle ORM migration 生成和执行。

## Risks / Trade-offs

- **[性能]** 大图加载可能较慢 → 使用原始 URL 加载高清图，显示加载指示器；弹窗打开时预加载下一张图片
- **[移动端]** 全屏弹窗在移动端体验可能受限 → 管理员审核主要在桌面端操作，暂不优化移动端
- **[数据迁移]** 新增字段不影响现有数据，`rejectionReason` 为可选字段，默认 null → 无需数据迁移
- **[缩放实现复杂度]** 自行实现缩放可能有边界 case → 限制缩放范围 0.5x-5x，做好边界处理

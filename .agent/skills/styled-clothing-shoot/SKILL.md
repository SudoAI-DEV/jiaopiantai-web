# 服装场景拍摄

## 概述

当用户希望生成 AI 服装电商图时，使用这个 skill。Agent 直接完成场景设计，然后调用渲染脚本输出图片。

**核心原则**：图片目标是**卖衣服**，不是展示风景。所有构图、光线、道具、姿态决策都必须服务于服装销售。

**硬性禁令**：`scenes.json` 绝对不能通过批量脚本、临时辅助脚本、固定模板、批量改字、成组复制 prompt 的方式生成。每款衣服都必须先实际查看该款服装图，再结合 `配置.yaml`、模特图和场景参考图，单独写出属于这款衣服自己的 10 个场景。

**配置职责**：`配置.yaml` 的首要维护责任在前置流程；本 skill 默认消费它。若当前产品目录缺失 `配置.yaml`，必须先补齐再继续。

## 工作流程

### 第 1 步：接收输入

用户通常提供：
- **产品目录**：`客户/{客户名}/{season}/{场景目录}/{产品编码}/`（以 `配置.yaml` 所在目录为根）。产品图直接放根目录，不使用 `原图/` 子目录。
- **客户要求与产品配置**：同目录下的 `配置.yaml`（client / season / scene / folder_requirements / selected_images / model_image / custom_requirements / image_descriptions / clothing_summary）
- **模特参考图**：`客户/{客户名}/models/`，以 `配置.yaml` 中 `model_image` 为准
- **场景参考图**：`客户/{客户名}/场景/{场景名}/`；旧项目回退到 `风格参考/`

### 第 2 步：分析图片并挑选产品图

先读取 `配置.yaml`。

**如果 `配置.yaml` 已包含 `image_descriptions` 和 `clothing_summary`**（由 `organize-product-folders` 预生成），直接使用文本描述，无需重新读取产品图片。仅在以下情况才重新看图：
- `配置.yaml` 缺少这些字段
- 客户补拍了新图但未更新分析
- 用户明确要求重新分析

**需要确认的信息**：
1. **客户要求**：client、season、scene、folder_requirements（来自 `配置.yaml`）
2. **服装信息**：品类、颜色、面料、廓形、图案、关键设计点（来自 `clothing_summary` 或看图）
3. **模特信息**：五官、身材、发型、整体气质（来自模特参考图，仍需查看）
4. **场景参考图**：色盘、影调、光线质量、构图习惯（仍需查看）
5. **客户细节图**：腰部、领口、袖口等区域必须作为近景重点

优先使用 `配置.yaml` 选定的服装图。默认精简，但产品存在正面/背面/下装/细节拆分时可保留 3-6 张，通过 `selected_image_notes` 和 `source_image_indexes` 明确职责。

#### 装饰元素清单（强制）

**如果 `配置.yaml` 已包含 `clothing_summary.decoration_elements`**，直接引用该清单，无需重新从图片分析。仅在清单缺失或信息不完整时才重新看图补充。

无论来源如何，最终清单必须包含每个元素的：

| 项 | 说明 |
|----|------|
| 名称 | 珍珠链、金属钉珠、装饰绳、宝石纽扣等 |
| 位置 | 仅正面 / 仅背面 / 两侧 / 腰部 / 领口等 |
| 形态 | 连续链状 vs 散珠、编织绳 vs 布带等 |

示例：
```
1. 珍珠链 → 仅正面领口沿边一圈，连续链状，背面无
2. 蓝色宝石纽扣 → 正面前中从腰线到裙摆，一列排列
3. 自裁腰带 → 正面系蝴蝶结，背面自然飘带
```

此清单直接指导 `clothing_description`、`required_details`、`back_only_details` 和 `forbidden_details`。标注"仅正面"的元素**严禁**出现在背面场景，反之亦然。

先用中文向用户汇报分析结果（含装饰元素清单），确认后继续。

### 第 2.5 步：同季同场景批量避重（批量任务强制）

连续生成同季同场景多款产品时，设计 `scenes.json` 前必须：

1. 读取同一 `season + 场景目录` 下最近 4-8 个兄弟产品的 `生成/batch_*/scenes.json`
2. 提炼已高频出现的 hero 场景家族、微场景、道具组合
3. 当前产品主动避开：
   - 与最近 3 个兄弟重复的前 10 张全身场景骨架顺序
   - 相邻 2 款重复同一微场景
   - 同一道具组合连续 3 款重复
4. 若某主场景必须沿用，至少把其余 3 个全身 + 2 个近景轮换到未高频使用的环境与道具

### 第 3 步：设计 10 个场景并写入 scenes.json

基于本文件规则，为该服装设计 10 个互不重复的场景。

**生成禁令**：严禁使用任何脚本产出 `scenes.json`；严禁从上一款复制后只改编码/颜色/面料名；尚未出图的旧 `scenes.json` 视为废弃草稿必须重写。已有正式生成结果的不重写，除非用户明确要求。

**设计要求**：
- 每个场景写出这款衣服独有的卖点，不允许空泛话术
- 场景差异来自衣服本身：版型、材质、正反结构、细节位置
- 近景跟随客户源图真实细节位置
- 批量任务中 `pose / lighting / background / model_direction / color_tone` 必须全部写满
- `scene_family` 变化不能只换名字却停留在同一视觉母体
- 对 `艺术建筑 (E)`：必须从 `references/scenes/scene-e-architectural-editorial.md` 的 20 个方案中选 1 个作为唯一主场景世界，`scene_01~08` 只在该建筑内部切换 2-3 个连通 micro-location；相邻兄弟 SKU 不应共享同一个方案编号
- 对 `自然田园 (B)`：必须从 `references/scenes/scene-b-country-garden.md` 的 `B01~B20` 中选 1 个，围绕同一庄园内部 2-3 个连通 micro-location 展开；相邻 SKU 不复用同一方案编号
- 写完前自检：遮掉产品编码，这 10 个场景仍能一眼看出是为这件衣服定制的
- **装饰元素交叉验证**：逐一核对清单，确保正/反面元素没有串位，所有元素至少在一个场景被展示
- **forbidden_details 检查**：每个场景至少含 2-3 条该衣服特有的易错项

**输出路径**：`客户/{客户名}/{season}/{场景目录}/{产品编码}/生成/batch_NN/scenes.json`，自动检测下一批次号。始终优先使用最新文件夹/版本。

### 第 4 步：渲染图片

```bash
cd /Users/orion/SudoAI/jiaopiantai
deno run --allow-read --allow-write --allow-env --allow-net \
  .agent/skills/styled-clothing-shoot/scripts/render.ts \
  -j "客户/.../{season}/{场景目录}/{产品编码}/生成/batch_NN/scenes.json"
```

单场景重渲染加 `--scene 3`。

### 第 5 步：复盘与迭代

检查质量并询问用户是否需要调整。若只需重做某场景，直接修改 `scenes.json` 中对应场景再单独渲染。

## 10 镜头框架

每个产品固定 **10 张**：**5 全身 + 5 近景**，严格 50/50。

### 全身镜头（`framing: "full_body"`）

| # | 镜头名 | 说明 |
|---|--------|------|
| 1 | 正面全身 | 自然带动作（整理配饰、迈步、轻微转身），展示正面穿着效果 |
| 2 | 背面全身 | 走开、回头、迎风而立，体现背面轮廓与线条 |
| 3 | 全身动态 | 行走、转身、与环境互动，表现垂坠感和动态 |
| 4 | 全身氛围 | 坐姿、倚靠、半蹲、放松停留，自然状态上身效果 |
| 5 | 全身环境互动 | 扶栏杆、穿门洞、靠墙、踩水边。场景与前 4 张不同 |

**全身构图硬性要求**：
- 全身镜头必须完整呈现模特**从头顶到脚底**，头顶上方必须留有空间（至少占画面 5%），严禁裁切头部、额头或脚部
- `full_prompt` 中必须明确写入"完整呈现头顶至脚底"等构图指令
- render.ts 会自动根据 `framing: "full_body"` 注入构图约束，但 prompt 本身也应包含

### 近景镜头（`framing: "close_up"`）

| # | 镜头名 | 说明 |
|---|--------|------|
| 6 | 上半身近景 | 裁切到腰/胸部。领口、肩线、袖型、面料纹理 |
| 7 | 下半身近景 | 裤型/裙摆、下摆线条、垂感、鞋履搭配 |
| 8 | 鞋履与下摆 | 裤脚/裙摆与鞋履关系。鞋型以最新参考图为基准 |
| 9 | 多角度细节拼图 | 2+1 或 1+2 多面板，2-3 个最值得卖的细节，面板间细白线分隔 |
| 10 | 正反面拼图 | 双面板正反面，同模特同场景同影调，允许面板内全身 |

**近景规则**：
- 不允许全身完整出镜（拼图面板内除外），清晰呈现服装细节
- 背景景深自然克制，禁止明显软件式大虚化；近景仍需保留可识别的环境纹理
- 若客户提供了最新拼图示意/参考版式，优先跟随
- 正反面拼图是**版式型销售图**：面板内允许全身，但整张图必须是拼图而非单张全身图

## 场景设计规则

### 自然感（最高优先级）

🚫 禁止证件照僵硬直站、夸张扭曲、正面僵硬直视、棚拍肖像感

✅ 每个姿态像"被自然抓拍的瞬间"：风吹发丝、半转身、整理衣物。视线多看远方/地面/侧面。身体重心自然，肩膀放松。整体像高端生活方式杂志抓拍。

### 影调与调色

每个场景必须写出具体的色温、反差、饱和度、光线质量、后期气质。`color_tone` 示例：`黄金时刻暖调，逆光轮廓光，低对比度胶片质感`，不要只写"自然光""柔和光线"。

### 背景复杂度

每个背景必须有三层深度：前景（虚化植物/岩石/道具）→ 中景（模特）→ 背景（环境叙事）。禁止纯色墙、大面积空沙滩、单平面背景。

### 背景人群控制

默认画面中只有主模特一人，不出现任何背景路人。场景类型间要拉开差异，至少覆盖 3-4 类环境。街拍优先选择可控干净外景，主动避开杂乱背景。

### 户外天气、鞋履、配饰

- 天气跟随场景参考：海边/建筑优先晴天直射光；田园优先明亮暖光；都市街拍优先晴天或明亮多云
- 鞋履以最新参考图为基准，不要机械统一成一种鞋型
- 配饰要丰富：耳环、帽子、墨镜、手提包、花束、咖啡杯等
- 连衣裙/半裙/裤装默认有口袋，至少 1-2 张展示手插口袋动作
- 所有服装默认有内衬、不透，禁止擅自做成透视效果

### 模特身份一致性

10 个场景必须同一模特。每个场景写 `model_direction`（表情情绪、发型配合、身体语言）。不要重新设计人脸。

### 画幅比例

统一 **3:4 竖版构图**。

### 字段说明

| 字段 | 说明 |
|------|------|
| `clothing_images` | `配置.yaml` 选定的服装图，2-6 张，有明确分工 |
| `source_image_notes` | 可选，按顺序说明每张源图角色（正面/背面/平铺），防混淆 |
| `product_config` | 配置.yaml 路径（唯一配置文件，含客户要求与视觉分析） |
| `batch_diversity_context` | 批量任务推荐，记录避重信息 |
| `scene_scenario` | 可选，`艺术建筑` 推荐写入方案编号如 `E07 修道院式回廊博物馆` |
| `framing` | 1-5 用 `full_body`，6-10 用 `close_up`（拼图也写 `close_up`） |
| `scene_family` / `micro_location` / `diversity_reason` | 批量任务推荐，标记场景锚点、具体落点、避重理由 |
| `pose` | 批量必填。重心、手部动作、视线方向、故事感 |
| `lighting` | 批量必填。光线方向、质量、时间感 |
| `background` | 批量必填。前/中/远景层次和环境材质 |
| `model_direction` | 批量必填。情绪、表情、身体状态 |
| `color_tone` | 批量必填。可执行的影调说明，不要写"高级感/杂志感" |
| `crop_focus` | 全身留空；近景明确区域；拼图必须描述面板布局 |
| `source_image_indexes` | 推荐。正面场景只喂正面图，背面只喂背面图，防串位 |
| `render_goal` | 可选，`validation` 验证结构，`final` 正式成片 |
| `required_details` | 当前镜头必须保留的细节，从装饰清单中挑选 |
| `front_required_details` | 正面场景必填，只含正面/两侧元素 |
| `back_only_details` | 背面场景必填，只含背面/两侧元素 |
| `bottom_required_details` | 下装可见场景必填 |
| `forbidden_details` | 必须含该衣服特有易错点，不能只用通用模板 |
| `seed` | 每个场景不同的 7 位随机数 |
| `full_prompt` | 100% 完整可直接发送 |

### clothing_description 编写规范

1. **来源必须是源图**：看不清写"难以确认"，不编造
2. **装饰位置精确到正/反面**：❌ "领口有珍珠链" → ✅ "仅正面领口沿边缀一圈连续珍珠链，背面无"
3. **区分前后差异**：前排扣后拉链、前钉珠后系带必须分别描述
4. **形态要具体**：❌ "珠子装饰" → ✅ "连续珍珠链" / "单颗金属钉珠均匀排列"
5. **10 个场景共用同一个**，必须同时覆盖正面和背面特征

### 关键镜头先验收

产品容易结构混淆时，先验收这 5 张：scene_01（正面）、scene_02（背面）、scene_06（上半身）、scene_07（下半身）、scene_10（正反面拼图）。

## Prompt 组装规则

每个 `full_prompt` 必须是完整自包含的艺术指导 prompt，渲染脚本直接发送给 Gemini API。

模板：

```
{brand_anchor}

图像角色规则：产品源图仅锁定服装。模特参考图锁定人物身份。最终人物必须与模特参考图是同一个人，保持一致的五官、脸型、发型、体型和气质。

模特一致性规则：同一模特的系列拍摄，所有场景中人物是同一个人。可调妆容和表情，面部身份不变。

姿态规则：自然松弛抓拍感，不要棚拍僵硬。视线自然，身体重心自然，有风感和动态。

影调规则：电影质感高级感。明确色温倾向、适当对比度、丰富光影层次。光线有方向感和质量感。

商业硬规则：不出现背景路人。有口袋的衣服要展示口袋。默认有内衬不透。

构图规则：{framing_rule}

本场景模特状态：{model_direction}

服装描述：{clothing_description}

拍摄场景：{scene_name}

{scene_prompt — 融合 pose/lighting/background/color_tone 的 100-150 字场景描述}

Output image resolution: {resolution}. Ultra high resolution, sharp details.

Negative: {negative}
```

`{framing_rule}`：
- **full_body**：`3:4竖版构图，人物全身入画，背景丰富有层次，明确呈现远景环境叙事。`
- **close_up**：`3:4竖版构图，近景/局部构图。{crop_focus}。默认只展示身体局部。若为拼图则写出多面板布局和分隔方式。服装细节必须清晰可见。背景适度虚化。`

`brand_anchor` 和 `negative` 使用标准值。

⚠️ 渲染脚本会自动追加 `required_details`、`front_required_details`、`back_only_details`、`bottom_required_details` 和 `forbidden_details` 作为额外约束块，因此 `full_prompt` 无需重复这些内容。

## 常见生成错误速查

| 错误类型 | 现象 | 防御方式 |
|----------|------|----------|
| 正反面装饰串位 | 正面装饰出现在背面或反之 | `clothing_description` 标注正/反面；`forbidden_details` 加入串位禁止；`source_image_indexes` 分流源图 |
| 装饰元素遗漏 | 装饰绳/流苏/钉珠在生成图中消失 | `required_details` 逐一列出；`clothing_description` 着重描述形态位置 |
| 装饰形态错误 | 珍珠链变散珠、装饰绳变缝线 | 用精确形态词："连续珍珠链"而非"珠子"；`forbidden_details` 加入形态纠正 |
| 版型/结构错误 | 领口变形、口袋消失、裙裤变半裙 | 精确描述领口类型和口袋位置；`forbidden_details` 写明版型混淆 |
| 花纹不匹配 | 印花被简化或色彩组合改变 | 详细描述图案元素和色彩组合 |
| 手部畸形 | 手指数量异常 | `forbidden_details` 含"畸形手""额外肢体"；避免复杂手部动作 |

- 每个 SKU 固定 10 张图

|------|------|------|
| `GEMINI_API_KEY` | 否 | 覆盖内置 API Key |
| `GEMINI_BASE_URL` | 否 | 覆盖内置 Base URL |
| `GEMINI_IMAGE_MODEL` | 否 | 覆盖图片模型（默认 `gemini-3-pro-image-preview`） |

# 四大场景模板定义

本文件现在作为 **场景总索引 + 通用规则 + 批量避重入口**。  
每个场景的具体场景家族、micro-location、光线和负面约束，已经拆到独立 markdown：

- `references/scenes/scene-a-seaside-art.md`
- `references/scenes/scene-b-country-garden.md`
- `references/scenes/scene-e-architectural-editorial.md`
- `references/scenes/scene-g-refined-nature.md`

`styled-clothing-shoot` 仍以 `SKILL.md` 中的 10 镜头规则为准；本文件负责帮助 Agent 快速判断场景边界、批量避重与优先阅读入口。户外天气必须跟随真实场景参考图，不要把所有外景都统一成烈日或阴天。

## 场景字段说明

- 当前客户新的参考入口是 `客户/{客户名}/场景/`，只保留 4 个核心场景：`海边艺术 / 自然田园 / 艺术建筑 / 大自然`。
- 产品目录、`要求.yaml` 与 `配置.yaml` 统一使用 `scene` 字段表示场景名。
- 若实际产品目录仍叫 `田园自然`，在参考层默认映射到 `客户/{客户名}/场景/自然田园/`；不要因此回退到旧的 7 风格逻辑。
- 旧的 `都市松弛 / 现代科技 / 室外街拍` 文档已归档，仅供历史项目参考；新任务默认不再把它们当成独立主场景。

## 通用补充规则

- 默认清场，不要在背景里生成路人、游客、外摆顾客或其他抢镜人物。
- 连衣裙、半裙、裤装优先保留口袋结构，并在合适镜头中随机安排自然手插口袋或拇指勾口袋边动作。
- 所有服装默认有内衬且不透；除非客户明确要求透明设计，否则不要生成透视薄纱感。
- 海景、田园、建筑和自然场景都要有更高端的空间材质与环境叙事，不要落成普通游客照。

## 批量任务防重总规则（适用于 `2026_春夏_第二批` 等同季多 SKU）

- 先看同一 `season + 场景目录` 下最近 4-8 个兄弟 SKU 已完成的 `生成/batch_*/scenes.json`，再写当前款。
- 当前款必须避开最近 3 个兄弟 SKU 的高频 `scene_family + micro_location` 组合，尤其是前 1-5 张全身镜头的 hero 场景骨架。
- 不要把“场景模板表”理解成固定答案；它只是边界和轮换池，不是每一款都要照抄同一组背景顺序。
- 若某款衣服因材质、长度或客户要求必须使用某个已出现过的主环境，至少轮换剩余 3 个全身镜头与 2 个近景落点。
- 推荐在 `scenes.json` 里填写 `scene_family`、`micro_location`、`diversity_reason`，方便下一款继续避重。

## 当前第二批重点场景轮换池

以下三组是当前仓库里 `客户/花姐&寅寅/2026_春夏_第二批/` 的重点场景。后续批量跑图时，优先按这些轮换池分散环境。

### 自然田园（对应产品目录常见命名：`田园自然`）

- 可轮换主环境：庄园草坪白伞区、碎石庭院桌景、温室花房入口、花房过渡廊道、白纱门廊、藤蔓旧门、花境小径 / 绿篱通道、庄园石阶、花园长椅 / 雕塑花器区、池塘草坪、水岸木栈道、果园 / 花树边、茶园边坡、石木院屋、乡野别墅门前、廊架外摆、复古黄车花园边、白栅栏草坡、喷泉庭院、湖畔草坪。
- 高危重复骨架：
  - `花房入口正面 + 花园小路背面 + 草地坐姿 + 藤蔓墙回眸`
  - `白纱门廊 + 藤蔓旧门 + 花园长椅` 连续复用
- 轮换优先轴：植物类型（花境 / 绿篱 / 果树 / 茶树 / 水边植物）、硬质结构（玻璃 / 石阶 / 碎石 / 围栏 / 石木门窗）、水体与否、座具类型（藤椅 / 长凳 / 躺椅 / 长桌）、光线状态（午前清亮 / 午后暖光 / 门廊柔光 / 廊架条状影）。
- 执行硬规则：
  - 先从 `scene-b-country-garden.md` 的 `B01 ~ B20` 里选 1 个主场景方案，再开始写当前 SKU。
  - 相邻兄弟 SKU 不要复用同一个主场景方案编号。
  - `scene_01 ~ scene_08` 优先停留在同一庄园 / 度假地世界内，只在 2-3 个逻辑连通的 micro-location 间切换。

### 艺术建筑

- 可轮换主环境：夯土美术馆前厅、方柱外廊、树影平墙前院、宽石台阶、暗廊望亮中庭、清水混凝土缓坡平台、修道院式回廊、光井中庭、低台庭院、屋顶露台、连桥廊桥、雕塑庭院、水池边、暖石长廊。
- 高危重复骨架：`门洞正面 + 楼梯背面 + 长廊行走 + 台阶坐姿`。
- 轮换优先轴：几何形状（弧 / 直 / 洞口 / 柱列）、材质（夯土 / 微水泥 / 混凝土 / 石材）、光源状态（硬朗日照 / 天窗柔光 / 树影切割 / 内外反差光）。
- 执行硬规则：
  - 先从 `scene-e-architectural-editorial.md` 的 20 个具体建筑场景方案里选 1 个，再开始写当前 SKU。
  - 相邻兄弟 SKU 不要共享同一个建筑场景方案编号。
  - `scene_01 ~ scene_08` 必须停留在同一个建筑世界里，只允许切换同一建筑 / 同一建筑群内部逻辑连通的 2-3 个 micro-location。
  - `scene_01 ~ scene_08` 必须写满 `pose / lighting / background / model_direction / color_tone`。

### 大自然

- 可轮换主环境：芦苇荡边缘、花海步道、草原坡地、高草坡顶、梯田观景点、湿地栈道、山野露台、民宿木平台、溪谷草坡、草甸山脊。
- 高危重复骨架：`普通草地背面 + 普通稻田正面 + 压暗黄昏站立 + 露台坐姿`。
- 轮换优先轴：地貌类型（芦苇 / 花海 / 草原 / 梯田 / 溪谷 / 露台）、景观精致度（花材密度 / 芦苇层次 / 草坡起伏 / 栈道结构）、阳光状态（晴天午后 / 金色侧光 / 明亮多云但有太阳方向）。
- 硬性禁令：禁止明显奶油化虚焦背景；禁止无日照的阴暗山野；禁止普通公园草地、路边树林、廉价郊游照式自然背景。

### 海边艺术

- 可轮换主环境：礁石海岸、潮汐池与海岸步道、热带棕榈屋、海边餐厅露台、海边艺术馆、棕榈庭院到海面过渡区。
- 高危重复骨架：`空沙滩正面站姿 + 海平线背面`、`同一块礁石反复拍正反两张`。
- 轮换优先轴：海岸类型（礁石 / 潮汐池 / 步道 / 建筑露台）、建筑类型（棕榈屋 / 餐厅 / 艺术馆）、时间段（清晨 / 午后 / 黄金时刻）。

## 相邻场景边界

- `自然田园`：偏庄园草坪、花境、白纱门廊、温室花房、池塘草坪、果园 / 茶园边、石木院屋与庭院桌景，是被整理过的庄园式自然。
- `大自然`：偏草海、梯田、山景、溪谷、湿地、观景位，是更大的山野旅行环境。
- `艺术建筑`：偏暖调夯土 / 微水泥 / 清水混凝土 / 弧墙 / 楼梯 / 几何洞口 / 树影。
- `海边艺术`：偏海岸叙事、礁石、海边建筑、潮汐池和热带度假感。

## 独立场景文件使用方法

- 在开始写某一款 `scenes.json` 前，先看本文件里的通用规则和当前场景的批量轮换池。
- 然后必须继续阅读对应场景的独立 markdown，再决定主场景家族与 micro-location。
- 如果是 `2026_春夏_第二批` 这种连续批量任务，优先把独立场景 markdown 里的“高危重复骨架”和“场景家族”当成避重依据。

# 认证与授权规格

## 认证方案

使用 **Better Auth** 进行用户认证。

### 支持的登录方式

1. **邮箱 + 密码**: 主要登录方式
   - 注册时需要填写基本信息（店铺名称、联系方式、主营类目）
   - 密码使用 bcrypt 加密存储

2. **手机号验证码登录** (可选)
   - 通过短信服务推送验证码
   - 预留扩展能力

### 角色定义

| 角色 | 说明 | 创建方式 |
|------|------|----------|
| customer | 客户，自助注册 | 用户注册 |
| admin | 管理员 | 管理员手动创建 |

## 权限控制

### 路由保护

- 未登录用户访问受保护路由 → 重定向到登录页
- 客户访问管理端路由 → 重定向到客户端首页
- 管理员访问客户端路由 → 允许访问管理端

### 数据隔离

- 客户只能查看和操作自己的产品和数据
- 管理员可以查看和操作所有数据
- 使用 Drizzle ORM + Middleware 实现数据层隔离

## API 端点

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/auth/sign-in` | POST | 邮箱密码登录 |
| `/api/auth/sign-up` | POST | 邮箱注册 |
| `/api/auth/sign-out` | POST | 登出 |
| `/api/auth/session` | GET | 获取当前会话 |
| `/api/auth/[...all]` | ALL | Better Auth 路由处理 |

## 安全考虑

- 密码使用 bcrypt 加密
- Session 使用 JWT
- API 路由使用中间件保护
- CORS 配置限制域名
